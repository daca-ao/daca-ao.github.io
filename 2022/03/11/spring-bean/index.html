Spring Bean

Posted on 2022-03-11

跟 EJB 一样，Bean 也是 Spring 应用中的核心组成部分。

有如下属性：

生命周期

主要包括以下方面：

  1. 构造实例化
  • 查找并加载需要被 Spring 管理的 bean，通过构造方法实例化 bean 对象

Bean 初始化概述：

  1. Spring 容器扫描 class（xml 配置文件）并解析，通过组装得到 BeanDefinition

  2. 根据得到的 BeanDefinition 生成 Bean

  3. 根据 class 推断构造方法

  4. 根据推断出来的构造方法，反射，得到一个对象（暂时叫原始对象）

  5. 填充原始对象的属性（依赖注入）

  6. 如原始对象中某个方法被 AOP 了，则需要根据原始对象生成一个代理对象

  7. 将最终生成的代理对象放入单例池（singletonObjects）中，下次 getBean 时直接从单例池取即可

  8. 属性赋值

  • 通过 setter 方法设置对象属性
  • 如 bean 实现了 BeanNameAware（实现 Aware 接口）：
    • 调用 Bean 的 setBeanName()
    • bean 初始化时调用
    • 传入从配置文件 xml 中得到的 ID（Bean ID）
    • 得到 BeanFactory 和 Bean 在 xml 中注册的 ID
  • 如 bean 实现了 BeanFactoryAware：
    • 则工厂调用 setBeanFactory(BeanFactory var1)
    • 将自身作为传入参数
  • 如 bean 实现了 ApplicationContextAware：
    • 调用 bean 的 setApplicationContext()
    • 引入 bean 所在应用的上下文
  1. 完成初始化
  • 如存在与 bean 关联的任何 BeanPostProcessors
    • 将 bean 实例传递给 BeanPostProcessor 中的 postProcessBeforeInitialization(bean, beanName)
    • 预初始化方法
    • 返回 Object
  • 如 bean 实现了 InitializingBean 接口：
    • Spring 调用 afterPropertiesSet()
  • 如 bean 使用 init-method 配置声明初始化方法：调用
  1. （前后处理）
  • 如存在与 bean 关联的任何 BeanPostProcessors
    • 将 bean 实例传递给 BeanPostProcessor 中的 postProcessAfterInitialization(bean, beanName)
    • 后初始化方法
    • 返回 boolean
  1. 使用后销毁
  • 如 bean 实现了 DisposableBean：
    • 调用 bean 的 destroy()
  • 如 bean 使用 destroy-method 配置声明销毁方法：调用

接口 BeanPostProcessor：

  • 定义 callback，允许在调用初始化方法前后对 bean 进行额外处理
  • 可实现多个 BeanPostProcessor
  • 设置 BeanPostProcessor 实现的 Ordered 接口提供的 order 属性控制执行顺序
  • ApplicationContext 会自动检测实现了 BeanPostProcessor 接口的 bean
    • 注册这些 bean 为 post processor
    • 适当时候调用
      import org.springframework.beans.factory.config.BeanPostProcessor;
      import org.springframework.beans.BeansException;

public class InitHelloWorld implements BeanPostProcessor {

public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
    System.out.println("BeforeInitialization : " + beanName);
    return bean; // you can return any other object as well
}

public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
    System.out.println("AfterInitialization : " + beanName);
    return bean; // you can return any other object as well
}

}
