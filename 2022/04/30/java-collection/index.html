Java Collection 接口

Posted on 2022-04-30 Views:

Collection 和 Map 接口并称 Java 集合类中的“二王”。

可将 Collection 理解为一个动态的对象数组；与数组不同的是，Collection 集合中的对象内容可以被任意扩充，方便于添加或删除。

特点是性能高，且容易扩展和修改。

* 注意 Collection 和 Collections 的区别。

子类及接口

AbstractCollection

  • 提供了 Collection 接口的部分骨干实现，为子类减少了实现 Collection 接口所需的工作
  • 直接子类：
    • AbstractList
    • AbstractQueue
    • AbstractSet

应用：不可修改集合

继承 AbstractCollection：实现一个不可修改集合

  • 实现 iterator 及 size 方法
  • iterator 方法返回的迭代器：iterator 接口的实现类，必须重写其 remove 方法
  • （AbstractCollection只声明 iterator 方法）
    public abstract Iterator iterator();
  • （且 iterator 方法被其他方法调用（如 contains 方法））
  • （Iterator 类的默认 remove 方法：）
    default void remove() {
    throw new UnsupportedOperationException(“remove”);
    }
  • 即：若不重写则抛出 UnsupportedOperationException 异常

应用：可修改集合

实现一个可修改集合

  • 实现 iterator、size 方法以及 add 方法
  • （AbstractCollection 默认的 add 方法）
    public boolean add(E e) {
    throw new UnsupportedOperationException();
    }
  • 即：若不重写则抛出 UnsupportedOperationException 异常

具体的非线程安全集合（部分）：

  • ArrayList：可动态增长和缩减的索引序列
  • LinkedList：可在任意位置高效增减的有序序列
  • ArrayDeque：用循环数组实现的双端序列
  • HashSet：没有重复元素的无序集合
  • TreeSet：有序集
  • EnumSet：包含枚举类型值的集
  • LinkedHashSet：可记住元素插入次序的集
  • PriorityQueue：允许高效删除最小/大元素的队列

List 接口

  • 继承自 Collection 接口
  • 有序集合，允许重复
  • 以线性方式存储元素：具有 索引 的概念
  • 可使用整数索引随机访问，或者列表迭代器获取对象
    • 默认构造产生 10 个元素大小的空 List
  • 直接实现类：
    • ArrayList
    • LinkedList
    • Vector
    • Stack

Set 接口

  • 继承自 Collection 接口
  • 访问其元素时为直接访问其对象的引用：不能有重复元素
  • 最多有一个为 null 值的元素

注：并没有声明 get()
具体如何访问 Set 中元素需要实现类定义

  • 子接口 SortedSet
    • 子接口 NavigableSet
      • 包含用于在有序集中查找和遍历的方法
  • 抽象实现类 AbstractSet
  • 通用实现类
    • HashSet
    • TreeSet

AbstractSet

  • 实现了 Set 接口，为 AbstractCollection 的子类
  • 添加了 hashCode() 和 equals() 的实现：用于对比两个集合的相等性
    hashCode()
  • 遍历集合中所有元素，返回它们的 hashCode 之和
  • 因此包含相同元素集的集合 hashCode 值应该相等
    equals()
  • 先比较两个集合的 size
  • 再调用 AbstractCollection 的 containsAll()，比较每个元素的相等性

SortedSet

  • 继承自 Set 接口
  • 暴露了用于排序的 Comparator 对象
  • 要求实现类必须保证元素有序：自然顺序或 Comparator 实现的排序方法
    • 且定义的 compare 方法可以获得集合的子集视图
  • 只为单个元素提供头部或尾部的访问：
    E first();
    E last();
    SortedSet headSet(E toElement); //获得子集视图，下同
    SortedSet tailSet(E fromElement);
    SortedSet subSet(E fromElement, E toElement);

NavigableSet

  • “可以导航的” set，SortedSet 的子接口
    lower(E s); //返回 set 中小于给定元素的最大元素
    floor(E e); //返回 set 中不大于给定元素的最大元素
    ceiling(E e); //返回 set 中不小于给定元素的最小元素
    higher(E e); //返回 set 中大于给定元素的最小元素
    pollFirst(); //获取并删除 set 中第一个（最小）元素
    pollLast(); //获取并删除 set 中最后一个（最大）元素

针对排序声明的方法：
NavigableSet descendingSet(); //返回由原 NavigableSet 反序排列的 NavigableSet
Iterator descendingIterator(); //根据 descendingSet() 结果再调用 iterator 方法

Queue 接口

  • 继承自 Collection 接口
  • 尾部添加元素，头部删除元素（FILO/LIFO）

Deque(/‘dek/, short for ‘double end queue’) 接口

  • 继承自 Queue 接口
  • 可在头部或尾部同时添加或删除元素，即允许在队列两边入队（offer）及出队（poll）
    • 实现类是 FIFO 队列时，元素被插在 Deque 尾部，头部元素先被删除
  • 不支持在队列中间添加元素
  • 实现类既可以是栈，也可以是队列

Queue 接口中声明的方法在 Deque 作为队列时相应的方法：
QueueDeque
add(e)
addLast(e)
offer(e)
offerLast(e)
remove()
removeFirst()
poll()
pollFirst()
element()
getFirst()
peek()
peekFirst()

Deque 的实现类也可作为 LIFO 的栈使用

  • 元素被插在 Deque 头部，头部元素最先被删除
    StackDeque
    push(e)
    addFirst(e)
    pop()
    removeFirst()
    peek()
    peekFirst()
  • 无论 Deque 作为队列还是栈使用，peek() 都返回头部元素
  • 与 List 接口不同，Deque 不提供对元素的索引访问
  • 虽然 Deque 允许插入 null 元素，但其实现类不应允许
    • 有时 null 是某些方法的特殊返回值
  • Deque 有两个通用实现类 LinkedList 及 ArrayDeque
    • 其实现类同时暴露 Deque 的 FIFO 和 LIFO 方法
    • 想要实现一个纯粹的队列或栈，还需亲自动手封装

具体实现类

ArrayList

链表
Java 中所有链表均为双向链接：每个节点还存放着指向前驱节点的引用
链表与泛型集合的区别：

  • 链表为有序集合，每个对象的位置十分重要
    • 将元素添加至链表中间时，需要依赖链表的位置；而迭代器负责描述集合中位置
    • 即：LinkedList.add() 由迭代器负责（只有对自然有序的集合使用迭代器添加元素才有实际意义）
  • 集合类库提供子接口 ListIterator ，包含add方法

应用：
LinkedList

TreeSet

  • 有序集合，使用红黑树完成排序
    • AbstractSet 子类
    • 实现了 NavigableSet
    • 增加了几个便于定位元素以及反向遍历的方法
  • 可以任意顺序将元素插入到集合中
  • 遍历时按照排序后的顺序输出
    • 每次添加元素，新元素会被放在正确的排序位置上
    • 迭代器总以排好序的顺序访问元素
    • 因此添加速度比散列表慢，但比添加到数组或链表快
  • 实现：Comparable 接口
    • 如果 a 与 b 相等，调用 a.compareTo(b) 返回0；如 a 大于 b ，返回值大于0；否则返回值小于0
  • 局限性：仅能实现一次
    • 因此可通过 Comparator 对象传递给 TreeSet 构造器，告诉 TreeSet 使用不同的比较方法
      public interface Comparator {
      int compare(T a, T b); // 返回值与compareTo 一样
      }
      需要比较的时候，可新建一个 Comparator 接口的类，将该类对象传入 TreeSet 构造器中；或使用匿名内部类方法传递。

注：如果要插入自定义的对象，就必须通过实现 Comparable 接口自定义排列顺序
Object 类中不提供任何 compareTo 接口的默认实现
注意数字比较时候的溢出：大正整数减去小负整数
覆盖 Comparator 中的 equals()，会使 TreeSet 添加元素的效率更高

  • 其他的 Set 实现类，如 HashSet、LinkedHashSet 详见 Map 接口

ArrayDeque

  • 继承自 AbstractCollection 类，实现了 Deque 接口
  • 内部实现是一个数组

优先队列

  • 元素按照任意顺序插入，按照排好序的顺序检索
  • 无论何时调用 remove 方法，总会获得当前优先队列中的最小元素
  • 使用堆（heap）实现，将最小元素移动到根节点
  • 优先队列既可保存实现了 Comparable 接口的类对象，也可以保存在构造器中提供比较器的对象
  • 典型示例：任务调度——每启动一个新的任务，都将优先级最高的任务从队列中删除

实现：Deque 队列与 Deque 栈
