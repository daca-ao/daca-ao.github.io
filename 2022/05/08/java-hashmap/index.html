HashMap

Posted on 2022-05-08 Views:

HashMap 在 Java 编程中的出镜率相当高。

这里涉及到一个基本的概念：

什么是 Hash？

直译为“哈希”，意译为“散列”，是根据 Hash 算法对某个对象进行序列化

散列集
散列表（hash table）：为每个对象计算一个整数，称为散列码（hash code）

  • 散列码由对象的实例域产生的一个整数，具有不同数据域的对象产生不同的散列码
  • 如果自定义类，需实现该类的 hashCode() 方法（该方法应与 equals 方法兼容）

Java 散列表：使用链表数组实现

  • 每个列表被称为“桶”（bucket）
  • 要想查找表中对象的位置，须先计算散列码，与桶的总数取余，所得为保存该元素的桶的索引
  • 有的时候会遇到桶被占用的情况——散列冲突（hash collision）
    • 需要将新对象与桶中的所有对象进行比较，查看该对象是否已经存在
    • 如有相同 Key 的元素：将其 value 更新
    • 否则：将该元素作为新的头元素加入桶中
    • 如散列码设计合理且随机分布，桶的数目足够大，比较次数也会少
  • 如想更多控制散列表运行性能，则需要指定一个初始的桶数：用于收集具有相同散列值的桶的数目
  • 如大致知道最终会有多少个元素要插入到散列表中，便可设置桶数：通常为预计元素个数的75%-150%
  • 若散列表太满：需要再散列（rehashed）
    • 创建一个桶更多的表，并将所有元素插入到新表中，再丢弃原来的表
    • 装填因子（load factor）决定何时对散列表进行再散列
      • 如设为0.75（默认值），而表中超过75%位置已填入元素：用双倍的桶数自动进行再散列

实现：
最简单的 set 类型：没有重复元素的元素集合

  • Java 集合类库提供 HashSet 类：实现基于散列表的集
  • 使用 add 添加元素
  • contains （被重新定义）快速查看某元素是否已在集中（只在某个桶中查找元素，不必查看集合中的所有元素）
  • 散列集迭代器依次访问所有桶。因散列将元素分散在表的各个位置上：访问顺序随机
  • 在不关心集合中元素顺序时才应使用 HashSet

HashMap / ConcurrentHashMap v.s. Hashtable

HashMap v.s. Hashtable
相近之处：

  • HashMap 和 Hashtable 均实现了 Map 接口
  • 均无序
    区别：
  • Hashtable 继承自 Dictionary 类；HashMap 继承自 AbstractMap 类
  • Hashtable 为 synchronized 的类，即线程安全类；HashMap 不是 synchronized 类
    • Hashtable 所有元素操作都是 synchronized 修饰的
    • 在单线程下 Hashtable 比 HashMap 慢
    • ConcurrentHashMap 为 HashTable 的替代
  • Hashtable 不可接受键或值为 null 的项；HashMap 可以接受 null 的键值对（key 和 value 为 null）
  • Hashtable 保留 HashMap 已经去除的 contains 方法
  • Hashtable 除了 HashMap 也有的 Iterator 实现方式，还保留了 Enumeration 方式
    • HashMap 提供 keySet 的遍历：fail-fast
      • 即：当其他线程改变了 HashMap 的结构，如增加删除元素，将抛出 ConcurrentModificationException
    • Hashtable 提供 key 的 Enumeration 遍历：不支持 fail-fast
  • 求 hashCode 的方法不同

// Hashtable 的 put：
public synchronized V put(K key, V value) {
if (value == null) {
throw …
}

int hash = key.hashCode();
...

}

ConcurrentHashMap v.s. Hashtable
相同：线程安全
不同

  • Hashtable 每次操作都会锁住整个表结构，导致每次只能有一个线程访问 Hashtable 对象
  • ConcurrentHashMap 只锁住某个节点，只有在涉及到 size 的操作才会锁住整个表结构

HashMap v.s. TreeMap

HashMap v.s. TreeMap

  • HashMap 在插入、删除和定位元素等操作占优
  • TreeMap 在遍历有序 key 集合时占优
