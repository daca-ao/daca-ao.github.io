Spring 注解

Posted on 2022-01-01

在 Spring IoC 概览中我们提到过，除了配置文件，Spring 还能通过注解进行 Bean 的依赖管理。

关于注解的配置也是在 Bean 的定义 XML 文件上，使用 <context:annotation-config/> 标记：

@Required

应用于 bean 的某个属性的 setter，标明该属性必须在配置文件 xml 中配置，且值不能为 null，否则容器会抛出 BeanInitializationException。

@Autowired

能应用于 bean 中某个属性的 setter 方法、非 setter 方法、构造函数和属性。

  • 注入一个 bean，完成自动装配
  • 可更精确地控制在何处和如何进行自动装配

注：以上配置文件没有配置 constructor-arg 或者 property，即非 xml 配置的 IoC。

默认情况：@Autowired 表明依赖是必须的（相当于标记了 @Required）

  • 可使用 @Autowired(required=false) 关闭默认行为

@Qualifier

  • 与 @Autowired 相结合
  • 指定具体的 bean 名称，从多个相同类型的 bean 中找出想要装配的某个 bean

JSR-250 注释

Java Specification Request，包括：
@PostConstruct：相当于 xml 配置中的 init-method
@PreDestroy：相当于 xml 配置中的 destroy-method

注：

  • 在配置文件中，该 bean 的 init-method 和 destroy-method 属性可无需配置
  • 如配置文件添加了 init-method 和 destroy-method：
    • 如配置的方法相同，则两者行为相同，两者互相覆盖
    • 如配置方法不同，则先执行注解的方法，再执行配置的方法

@Resource

  • 在 setter 中使用
  • 使用 name 属性，遵循 byName 语义自动装配
  • 如没有指定 name：使用字段名或属性名称

注：

  • @Resource 和 @Autowired 功能是一样的，都需要配置 DI 注解解析器
  • @Resource 是 JavaEE 的注解，默认按名称注入；@Autowired 是 Spring 的注解，默认按类型注入

@ComponentScan：扫描指定包下的所有 Spring 组件
@Bean：手动创建一个 Spring Bean
@Primary：手动创建 Spring Bean 时，指定当前为默认注入的 bean
@Scope：配置 Spring bean 的作用域
@Lazy：表示延迟注入 bean

@Component

  • 将 Java 类标记为 bean
  • 是任何 Spring 管理组件的通用构造型
  • Spring 组件扫描机制现在可以将其拾取并将其拉入应用程序中
    @Controller
  • 将 Java 类标记为 Spring Web MVC 控制器
  • 标记了它的 bean 会被自动导入到 IoC 容器中
    @Service
  • @Component 注解的特化
  • 不会对 @Component 注解提供任何其他行为
  • 可在服务层类中使用 @Service 而不是 @Component
    @Repository
  • @Component 注解的特化
  • 将 DAO 导入 IoC 容器，并使未经检查的异常有资格转换为 Spring DataAccessException
