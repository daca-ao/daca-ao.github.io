网络层与 IPv4

Posted on 2023-11-17 Views:

聊一聊 OSI 模型中的第三层：网络层。

网络层通过路由选择算法提供路由和寻址的功能，为 IP 分组从源主机到目的主机选择一条合适的传输路径，并为下一层 —— 传输层提供端到端的数据传输服务；同时网络层具有一定的拥塞控制和流量控制的能力。

符合网络层功能，专门处理网络层路由与转发功能的计算机，我们称之为“路由器”。

由于 TCP/IP 协议体系里面的网络层功能由 IP 协议规定和实现，因此网络层又称为 IP 层。

那，什么是 IP 分组？

IPv4

在网络层中，一个完整的网络地址包括：

  • 网络名字（name）
  • 网络地址（address）
  • 路径（route）

IP 地址标识的是一台主机或路由器与网络的接口，最早由 1981 年的 RFC791 描述了 IPv4 协议。

IP 协议的主要特点如下：

1. IP 协议是无连接、不可靠的分组传送服务协议

  • 不提供对分组传输过程的跟踪，意为“尽力而为（best-effort）”；
  • 无连接：指的是 IP 协议并不维护 IP 分组发送后的任何状态信息，每个分组在各自的传输过程中相互独立；
  • 不可靠：指的是 IP 协议不能保证每个 IP 分组都能正确地、不丢失和按顺序地到达目的主机；

IP 协议设计重点应放在系统适应性、可扩展性与可操作性上，在分组交付可靠性上只能做一定牺牲。

2. IP 协议是点-点（Peer-to-Peer）的网络层通信协议

  • 网络层需要在 Internet 中为通信的两个主机之间寻找一条路径，该路径通常由多个路由器、点-点链路组成；
  • IP 协议要保证数据分组从一个到另一个路由器，能通过多条路径从源主机到目的主机。

因此，IP 协议是针对源主机-路由器、路由器-路由器、路由器-目的主机之间的数据传输的点-点线路的网络层通信协议。

3. IP 协议屏蔽了互联的网络往下在数据链路层、物理层协议与实现技术上的差异

  • 各种网络和协议，如广域网、城域网或局域网，都有可能是异构的；
  • 即使都是局域网，其物理层、数据链路层协议也可能不同；
  • 通过 IP 协议，网络层向传输层提供统一的 IP 分组，传输层不需要考虑互联网络在数据链路层、物理层协议与实现技术的差异。

IPv4 分组（IPv4 数据报）格式

回答上面的问题：一份 IP 分组，就是一次 IP 报文的发送。

一份 IPv4 的报文由两个部分组成：

  • 分组头：也称“首部”，长度可变。习惯用 4 字节（8 位）为基本单元表示分组头字段
    • 前 5 行（共 20 字节）为每个字节头中必须有的字段
    • 第 6 行开始为选项字段
    • 因此 IPv4 分组头基本长度是 20 字节 —— 如加上最长 40 字节（10 行）的选项（options），则 IP 分组头最大可达 60 字节
  • 数据

分组头格式

分组头总览如下：

  1. 版本（version）：IP 分组第一个字段，4 位
  • 表示所使用的网络层 IP 的版本号
  • 字段值为 4，表示 IPv4；6 则表示 IPv6
  • IP 软件在处理该分组之前必须检查版本号，以避免错误解释分组的内容
  1. 分组头长度（header length，也称报头长度）：4 位，定义以4字节为一个单位的分组头的长度
  • 分组头出了 IP 选项字段和填充字段外，其他各项为定长；
  • 因分组头基本部分为 20 字节，故其字段最小值为 5（5 × 4 = 20）
  • 一个有 IP 选项（options，在可选部分中）字段与填充（padding，在可选部分中，不一定有）字段的 IP 分组的头部大于 20 字节
  • 协议规定
    • 分组头长度字段最大值为 15（15 × 4 = 60）
    • IP 分组头长度必须是 4 字节的整数倍，如不是 4 字节整数倍，则有填充字段“补0”
  1. 服务类型（differentiated service）：8 位，指示路由器如何处理该分组
  • 4 位服务类型 TOS 字段
  • 3 位优先级 precedence 字段
  • 1 位保留位

3.1. 服务类型字段（Type of Service, TOS）：4位，分别为4个参数，每位表示

  • D（Delay，延迟）
  • T（Throughput，吞吐量）
  • R（Reliability，可靠性）
  • C（Cost，成本）
    每位的取值为0或1，0在每一位都表示正常级（normal）
    每个组合的服务类型参数的4位中，最多只能有一位的值为1，其他三位的值则为0
  • 延迟位 D = 1 表示 low delay；D = 0 表示 normal
  • 吞吐量位 T = 1 表示 high throughput；T = 0 表示 normal
  • 可靠性为 R = 1 表示 high reliability；R = 0 表示 normal
  • 成本为 C = 1 表示 low cost；C = 0 表示 normal
    每种网络传输技术都不可能同时在延迟、吞吐量、可靠性和成本4个方面达到最优——必须寻求折中方案，保证最基本性能

3.2. 优先级（precedence）：3位

  • 分组在网络间传输时：有的应用需要网络提供优先服务，重要服务信息的处理等级比一般服务信息的处理等级高
  • 当网络高负荷，路由器发生拥塞必须丢弃一些分组时：路由器将只接收某优先级以上的分组
  • 路由器根据四个参数的组合来确定选择的标准
  1. 总长度（total length）字段：16 位，定义以字节位单位的分组总长度
  • 分组头长度与数据长度之和
  • 能表示IP分组最大长度为65535（216-1）字节，其中包括分组头长度
  • IP分组中高层协议的数据长度等于分组的总长度减去分组头长度
  1. 标识字段（identification）：16 位
  • 最多可分配 65535 个ID
  • 是 IP 分组的片（fragment）识别的标识
  • 应用场景：
    • 属于同一分组的不同片到达时会出现乱序，或与属于其他分组的片混在一起
    • 目的主机根据标识字段的 ID 值（如1562），将所有同样 ID 值的片挑出来，重新组装成一个分组
  1. 标志（flags）：3位
  • 最高位是0，该值必须复制到所有分组中
  • 不分片（Do not Fragment, DF）值：必须复制
    • DF = 1：接收主机不能对分组分片
      • 如分组长度超过 MTU 又不可分片：该分组只能被丢弃，并要用 ICMP 差错报文向源主机报告
    • DF = 0：接收主机可以分片
  • 分片（More Fragment, MF）值
    • MF = 1：接收的分片不是最后一个分片
    • MF = 0：接收到的分片为最后一个
  1. 片偏移（fragment offset）：13位
  • 表示分片在整个分组中的相对位置

  • 以 8 字节为单位计数：选择的分片长度应为 8 字节的整数倍
    例：

  • 分组头长度 20 字节，MTU 长度为 820 字节

  • 第一片：包括 0-799 编号数据
    • 复制原分组头（除去标志和片偏移值），与第一分片数据构成第一分片
    • 初始分片，片偏移值为 0
  • 第二片：包括 800-1599 编号数据
    • 复制原分组头（除去标志和片偏移值），与第二分片数据构成第二分片
    • 该分片第一个数据编码号为 800，800/8 = 100，因此偏移值为 100
  • 第三片：包括 1600-2199 编号数据
    • 复制原分组头（除去标志和片偏移值），与第三分片数据构成第三分片
    • 该分片第一个数据编码号为 1600，1600/8 = 200，因此偏移值为 200

从原始分组到分片后，分组头总长度字段、标志和片偏移字段均发生变化

  • 分片1、分片2中 MF = 1；分片3中 MF = 0
    因标识、标志与片偏移均发生变化，分组头校验和需重新计算
  1. 生存时间（time-to-live, TTL）：8位
  • IP 分组从源主机到目的主机的传输延迟不确定：如出现路由器的路由表错误，会造成分组在网络中循环、无休止流动
  • IPv4 设计该字段，用来设定分组在 Internet 中的“寿命”
  • 通常用转发分组最多的路由器跳数（hop）来度量
    • 初始值由源主机设置，每经一路由器转发，TTL 值减一
    • 当 TTL = 0，丢弃分组并发送ICMP报文通知源主机
  1. 协议（protocol）字段：8位，指使用 IP 的更高一层（传输层）协议类型

常见协议类型如下：

协议字段值 高层协议类型
1 ICMP
2 IGMP
6 TCP
8 EGP
17 UDP
41 IPv6
89 OSPF
  1. 头部校验和（header checksum）：16位
  • 保证分组头部的数据完整性
  • IP 分组只对分组头进行校验而不包括分组数据
    • IP 分组头之外的部分属于高层数据，高层数据都会有相应的校验字段：因此 IP 分组可以不对高层数据进行校验
    • 由于每经过一个路由器，IP分组头都要改变一次，但数据部分不改变：设置头校验和只对变化部分进行校验是合理的，可减少路由器对每个接收分组的处理时间，提高路由器运行效率
      计算方法：二进制反码求和
  1. 将 IP 头看成 16 位字组成的二进制 bit 序列，计算前将校验和字段置 0
  2. 对 16 位字进行求和运算：如最高位出现进位，则将进位加到结果的最低位
  3. 将最终求和结果取反，得到校验和
    检错能力不强，但算法简洁，运算速度快
  1. 地址字段
  • 源地址（source address）
  • 目的地址（destination address）
  • 均为32位
  • 最简单部分，分组整个传输过程中保持不变
  1. 分组头选项（options）
  • 主要目的：用于控制和测试

  • 注意：

    • 用户可以不使用该选项，但作为 IP 分组头的组成部分，所有实现 IP 的硬件或软件都应能处理它
    • 最大长度为 40 字节——如用户使用的选项长度不是 4 字节整数倍：添加补充位补足
    • 分组头选项组成：
      • 选项码：用于确定该选项具体功能——如源路由、记录路由、时间戳等
      • 长度：表示选项数据的大小
      • 选项数据

  • 源路由：指由发送分组的源主机制定的传输路径，用来区别由路由器通过路由选择算法确定的路径

    • 主要用于测试某网络的吞吐量，绕开出错的网络
    • 也用于保证分组传输安全的应用
    • 严格源路由（Strict Source Route, SSR）
      • 规定分组要经过的路径上：
        • 相邻路由器间不能插入其他路由器
        • 经过的路由器顺序不能改变
      • 主要用于网络测试
        • 将第一个测试点地址设定为分组头中的目的地址
        • 将最后一个测试点主机地址设定为路径数据字段中最后一个指定地址
    • 松散源路由（Loose Source Route, LSR）
      • 规定分组一定要经过的路由器，但不是一条完整的传输路径
      • 中途可经过其他路由器
  • 记录路由（Record Route）：
    • 将分组经过的每个路由器 IP 地址记录下来
    • 常用于网络测试
      • 如网管要了解发送到某个主机的分组经过哪些路由器才能到达目的主机
      • 以及互联网中的路由器配置是否正确
  • 时间戳（Timestamp, TS）
  • 记录分组经过每个路由器的本地时间
  • 采用格林尼治时间（毫秒）
  • 利用其追踪路由器运行状态，分析网络吞吐率、拥塞情况与负荷情况等

IP 分组的分片与组装

  • 相关字段：标识（identification）、标志（flags）与片偏移（fragment offset）
  • 从IP与数据链路层协议角度看：
    • IP分组作为网络层数据，必然通过数据链路层，封装成帧再通过物理层传输；
    • 一个分组可能经过多个不同网络，每个路由器都要将接收的帧进行拆帧和处理，然后封装成其他类型的帧；
    • 帧的格式与长度取决于网络采用的协议，每种协议对帧的格式、长度的要求不同
      • 一路由器从一Ethernet接收到一个帧，再转发到一个Token Ring网
      • 路由器接收的是Ethernet帧，转发时需要封装成Token Ring帧
    • 每种局域网规定：帧的数据字段最大长度称为最大传输单元（Maximum Transfer Unit, MTU）
    • 不同网络的MTU长度不同
    • RFC791 规定：IP分组最大长度为 65535（216-1） 字节
  • 从IP与传输层协议的角度看：
    • 传输层数据包必须在网络层封装成IP分组，再传送到数据链路层组成帧
    • 封装IP分组时，传输层数据包加上IP分组头的总长度必须小于 65535 字节
      • 因此：设计时从应用层和传输层开始控制报文长度，避免被分成多个分组的问题
  • IP 分组通常需被分成若干较小的片（fragment）传输
    • IP 分组最大长度（65535 bytes） > 实际使用网络的 MTU （Ethernet: 1500B）
    • 传输路径中需要连接多个网络，不同网络的 MTU 长度可能不同
    • 路由器根据下一跳 MTU 大小，决定该分组转发前是否需要分片

基本方法：

  • 首先确定片长度
  • 后将原始IP分组包括分组头分成第一片
  • 如剩下数据仍超过片长度：进行第二次分片
  • 直至剩下数据小于 MTU 为止

6.3.5. 网络地址转换（Network Address Translation, NAT）

并没有深层次解决 IPv4 地址严重不足的问题

演进：IPv6

早期设计的 IP 分组结构、IPv4 地址、网络层服务质量 QoS 都不能满足 Internet 大规模发展的要求

  • IPv4 协议中对于分组结构与分组头结构的规定不变
  • 变化的部分包括三个方面：
    1. IP 地址处理方法
    2. 分组交付的路由算法与路由选择协议
    3. 如何提高协议的可靠性、服务质量与安全性

2011 年 2 月 3 日，最后 5 块 IPv4 地址被分配完毕——需要新的 IP 协议（IPv6）
