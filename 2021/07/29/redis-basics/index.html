Redis 入门

Posted on 2021-07-29

Redis，其实是 REmote DIctionary Server （暂译为“远程字典服务”吧，虽然有些拗口）首字母的缩写。

优势：

  • 性能高
  • 数据类型丰富，支持五种对象类型：String, List, Hash, Set, Sorted Set(zset)
  • 所有操作都是原子性的
  • 单个操作为原子性，多个操作支持事务
  • 支持发布与订阅、通知、key 过期等特性

EXPIRE key seconds // 为 key 设置过时时间，以秒计算

EXPIREAT key timestamp // 设置 key 的超时时间至某个时间戳

单进程单线程的模型

  • 完全基于内存的操作，CPU 不是瓶颈
  • 单线程容易实现
    为何仍然这么快：
  • 数据存储类似于 HashMap：查找和操作的时间复杂度是 O(1)
  • 数据结构简单，对数据操作也简单
  • 采用单线程，避免不必要的上下文切换和竞争条件
  • 使用非阻塞 IO

Redis 是基于键值对的数据库，其中“值”的类型有：

String
概况：

  • Redis 最基本的类型，长度不超过 512 MB
  • 二进制安全，可以包含任何数据，如 jpg 图像或者序列化的对象
    编码：
  1. int：8 个字节的长整型
    1. 字符串值为整型时，该值使用 long 整型表示
  2. embstr：<= 39 字节的字符串
  3. raw：大于 39 字节的字符串
    注：
  • embstr 与 raw 都使用 RedisObject 和 SDS 保存数据
  • embstr 的使用只分配一次内存空间（RedisObject 和 SDS 连续）
    • 好处：创建时少分配一次空间，删除时少释放一次空间；对象所有数据连在一起，寻找方便
    • 坏处：字符串长度增加需要重新分配内存时，整个 RedisObject 和 SDS 都需要重新分配空间
    • 因此 embstr 实现为只读
  • raw 需要分别为 RedisObject 和 SDS 分别分配内存空间

编码转换：
当 int 数据不再是整数，或大小超过 long 范围：自动转为 raw
因 embstr 的实现是只读：修改时先转化为 raw 再进行修改

Hash
键值对（key-value）集合
本质为 String 类型的 field 和 value 的映射表
指令以 h 开头

List
概况：
简单的 String 列表，按照插入顺序排序
可添加至列表头（左）或尾（右）
指令以 l 开头

Set
String 无序不重复集合，通过哈希表实现
指令以 s 开头

zset（Sorted Set）
String 有序不重复集合，与 Set 类似
每个元素关联一个 double 类型的分数，Redis 通过分数从小到大排序
指令以 z 开头

每种对象类型都有至少两种编码：

  1. 接口与实现分离，当需要增加或改变内部编码时，用户使用不受影响
  2. 可根据不同应用场景切换内部编码，提高效率

Redis 作为内存数据库，在内存中存储内容主要是数据（键值对）

内存统计：
$info memory

Memory

used_memory:809936
used_memory_human:790.95K
used_memory_rss:7909376
used_memory_peak:809936
used_memory_peak_human:790.95K
used_memory_lua:35870
mem_fragmentation_ratio:9.77
mem_allocator:jemalloc-3.6.0

used_memory

  • Redis 分配器分配的内存总量（Byte）
  • 包括使用的虚拟内存（swap）

used_memory_human

  • used_memory 更友好的显示方式

used_memory_rss

  • Redis 进程占据操作系统的内存（Byte），与 top 和 ps 命令看到的值一致
  • 包括：
    • Redis 分配器分配的内存
    • 进程运行本身需要的内存
    • 内存碎片
    • ……
  • 不包括虚拟内存

used_memory 与 used_memory_rss 比较

  • 前者为从 Redis 得到的量
  • 后者从操作系统角度得到的量
  • 两者不同的原因：
    1. 内存碎片和 Redis 进程运行需要占用内存：前者可能比后者小
    2. 存在虚拟内存：前者可能比后者大

mem_fragmentation_ratio

  • 内存碎片比率，used_memory_rss / used_memory
  • 实际应用中 Redis 数据量比较大：进程运行占用内存与 Redis 数据量和内存碎片相比会小很多
  • 一般大于 1，值越大内存碎片比例越大
  • 小于 1，说明 Redis 使用了虚拟内存：要及时排查
  • 1.03 为比较健康的状态

mem_allocator

  • Redis 使用的内存分配器，编译时指定
  • 可以是 libc, jemalloc 或 tcmalloc，默认 jemalloc
    jemalloc 将内存空间划分为小、大、巨大三个范围
    每个范围内划分许多小的内存块单位
