Java 中的锁实现

Posted on 2021-08-22

锁是控制多个线程访问共享资源的方式，是实现多线程同步的重要思想。java.util.concurrent 包中有很多相关的 API 和组件。

一般来说，锁能够防止多个线程同时访问一个共享资源；个别锁允许并发访问共享资源，如读写锁等。

本帖简单来介绍一下这些 API 和组件的实现细节和使用方式。

Lock 接口

在 Lock 接口之前，Java 使用 synchronized 关键字隐式地获取和释放对象同步锁。
在随后的 Lock 接口中，虽然使用的时候需要显式地获取和释放锁，没有 synchronized 来得便捷；但是 Lock 增加了获取和释放锁的可操作性、可中断地获取锁、超时获取锁等功能，使用更灵活，扩展性更强：

特性 描述
尝试非阻塞地获得锁 当前线程尝试获取锁，如果此刻锁没被其它线程获得，则成功获取并持有锁
能被中断地获取锁 与 synchronized 不同：获得锁的线程能够响应中断，当获取到锁的线程被中断时，中断异常会被抛出，同时锁会被释放
超时获取锁 在指定时间内获得锁，如果截止时间到了依旧无法获得锁，则返回

使用：

以上虽然也是 synchronized 的原理解释代码，但是 synchronized 是上述这一套加锁解锁的固化，而单纯使用 Lock 接口能更灵活地实现业务。

ReentrantLock

可重入锁：线程可重复获得已经持有的锁

  • 锁保持持有计数（hold count）：跟踪对 lock() 的嵌套调用
  • 被一个锁保护的代码可调用另一个使用相同锁的方法
    • 如 Block A 中代码调用 method B，B 与 A 持有相同的锁对象，则锁的持有计数计为 2
    • 调用 B 执行完后，计数计为 1

基本结构：

  • 确保任何时候只有一个线程进入 critical section
  • 当其他线程调用 lock() 时将会被阻塞，直到第一个线程释放该锁对象
  • 锁不能使用在 try-with-resource 语句
    • “解锁”的方法名不是 close
    • try-with-resource 在使用时希望声明新变量，而锁是多个线程共享的变量，不能随便新建
  • 注：如在临界区之前抛出异常，finally 释放锁，但数据可能会受损

Condition 接口

  • 某些线程进入临界区 critical section 后，还需要满足一定条件才能执行
    • 注：不能在多线程中简单使用判定后读写的方法——有可能在 if 判断后被中断
  • 使用 Condition 对象管理已获得锁但不能做有用工作的线程

注：Condition 方法的命名（await(), signalAll(), signal()）分别与 Object 同样功能的 wait()、notifyAll()、notify() 区分开

关键点：

  • 锁用来保护代码片段，任何时刻只能有一个线程执行被保护的代码
  • 锁可管理试图进入被保护代码段的线程
  • 锁可拥有一个或多个相关条件对象
  • 每个条件对象管理已进入被保护代码段但未能运行的线程

注：锁和条件并非面向对象

监视器（Monitor）

  • 在 JVM 堆中，每个对象占有一块空间
  • 其中一部分称为对象头
  • 对象头有一组字段用于表示锁的状态，这组字段为 Monitor
    • 包含 线程持有者 + 程序计数器
    • 计数器为 0 时锁开启：线程先到先得

监视过程：

  • 线程获取某对象的锁后：JVM 记录线程信息，计数器 + 1
  • 非持有锁的线程访问该对象 synchronized 方法或代码块，且计数器 > 0：线程等待
  • 锁持有线程再次访问该对象 synchronized 方法或代码块：拿到同样的锁，计数器 + 1
  • 线程退出 synchronized 方法或代码块：计数器 - 1，直至减到 0，锁被释放
  • 锁有任意多相关条件

cas
让 CPU 比较内存中某个值是否和预期的值相同，如果相同则将这个值更新为新的值，不相同则不做更新
CAS 的读写都具有原子性，实现方式是通过 C/C++ 调用 CPU 指令完成的

ABA
两个线程同时对同一个值进行 CAS 操作，初始值为 A，要更新到 B
线程 1 抢先获得时间片，然后线程 2 阻塞，线程 A 发现与期望 A 值相等，替换为 B
在线程 2 恢复之前，线程 3（或者继续是线程 1）过来进行 CAS 操作，期望值为 B，更新到 A；因为条件刚好符合，所以值被更新回 A
线程 2 恢复后，发现初始值和期望值相同，又将值更新到 B

虽然线程 2 完成了更新的操作，但是它并不知道值从 A->B->A 的过程
